How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area places files to be committed to the repo from the working directory in a storage area to be labeled.  It offers a clean way to update without including everything, potentially confusing and bloating updates. 
_________________________________________________________________________________________________________________________________________________________________________________
How can you use the staging area to make sure you have one commit per logical change?

It allows you to control and see which changes you will make before committing.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

Branches allow you to make changes without impacting a working master branch base of code.  These changes can be mreged over time if satisfactory.
